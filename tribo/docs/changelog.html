      • M = Major Version
      • m = Minor Version
      • R = Release
      • B = Build
    Obs.: Essas informações podem ser obtidas cadastrando-se o Sistema Monitorado na opção Manutenção/Cadastro/Perfís Sistemas e, nos campos "Identificador de Versão/Configuração:", selecionando-se o "ítem Nome/Seção/Chave de Arquivo INI", digitando-se no campo seguinte a informação "Cacic\cacic2.ini/Coleta/te_versao_xxx_yyyy", onde "xxx_yyyy" equivale ao nome do módulo.
    Exemplo: "Cacic\cacic2.ini/Coleta/te_versao_col_hard" (retornará a versão do coletor de informações de hardware)

30/09/2004

Com o total de 700 computadores em seu banco de dados, o CACIC/ES tem se mostrado estável e tem sido enriquecido com alguns novos recursos:

  • Criação de 4 níveis de acesso:
    • Administrador Master: Administração plena do sistema;
    • Comum: Acesso limitado aos relatórios, estatísticas e detalhamentos de informações de computadores;
    • Convidado: Para acesso de usuários temporários e/ou visitantes, o qual não visualizará opção de Troca de Senha e alguns relatórios;
    • Gestor de Patrimônio: Para usuários da área de gestão patrimonial, os quais terão à disposição um recurso para "Forçar Coletas" de informações patrimoniais em computadores específicos;
  • Mecanismo de filtro de endereços MAC coletados nas estações: devido à possibilidade de incidência de vários protocolos e/ou dispositivos diferentes de TCP/IP e ETHERNET (PPP, vmWare, DialUp, Placa TV/Internet, Discadores proprietários, etc.);
  • Opção Administrativa "Forçar Coletas": para que o usuário com nível "Gestor de Patrimônio" possa requisitar o preenchimento de dados patrimoniais em uma determinada estação;
  • Updates de Sub-Redes: onde o Administrador Master poderá comandar a atualização nos servidores de updates para padronização das versões (atualização de agentes nas estações);
  • Relatório com informações de "Variáveis de Ambiente": para rastreamento dos valores atribuídos às variáveis de ambiente nas estações, por exemplo: GIRAFA, CLIPPER, ESTACAO, IMPSAL, etc.
  • Relatório com informações de "Inventário de Softwares": para mapeamento dos softwares instalados nas estações gerenciadas.

17/09/2002

Liberada a versão 0.0.4.1 do agente Windows do CACIC. Essa versão instala automaticamente o Windows Socket 2 Update (Winsock 2) no Windows 95, caso este update ainda não esteja instalado. É importante lembrar que versões do CACIC iguais ou superiores à versão 0.0.4.0 requerem o Winsock 2 para funcionarem corretamente. Versões do Windows superiores à versão 95 já têm suporte nativo ao Winsock 2.

Incluída a unidade de medida usada nas colunas referentes à utilização de espaço em disco no módulo web de informações sobre os servidores SCO Unix. Colaboração de Mery Torres (ESES.R).

Atualizado o manual de instalação, no que diz respeito à configuração do proftpd. Para o perfeito funcionamento da rotina de autoupdate de versões do agente Windows do CACIC superiores ou iguais à versão 0.0.4.0, é necessário que o diretório /home/cacic/bin esteja configurado da seguinte forma, no arquivo /etc/proftpd.conf:
<Directory /home/cacic/bin>
   <Limit READ CWD>
      AllowAll
   </Limit>
   <Limit STOR RMD DELE MKD>
      DenyAll
   </Limit>
</Directory>

Colaboração de Jarbas Peixoto (ESGO.P).

Liberada a versão 0.0.4.0 do agente Windows do CACIC. As novidades dessa nova versão são as seguintes:

  • Coleta da versão do antivírus VirusScan;
  • Melhoria na rotina de autoupdate do agente Windows;
  • Atualização da rotina de detecção da versão do Windows (Windows 2000 e Windows XP). Colaboração de Luís Leite (DBAO.R);
  • Adicionada a opção de inclusão e exclusão de chaves do registro (registry) do Windows a partir do módulo web de configuração centralizada dos agentes, dando grande flexibilidade ao administrador na manipulação do registro das estações;
  • Corrigido erro de digitação na mensagem exibida durante a instalação do agente Windows ("sempre que o windows foi iniciado" -> "sempre que o windows for iniciado"). Colaboração de Anderson Peterle;
  • Incluída a opção "Desabilitar Conexão Automática" no RPrint. Essa opção instrui o agente Windows do CACIC a desabilitar automaticamente as opções "Conexão Automática", "EBCDIC", "Edita Mensagem" e "Remove NULL" do RPrint, sempre que houver uma impressora sem nome cadastrada no Plenus. Dessa forma, não são geradas entradas em duplicidade com o nome "IMP" nos acessos aos mainframes. Colaboração de Romeu Scárdua (ESES.R) e Heleno Guilhon (ESES.R).

Feitas atualizações nos módulos Web.
